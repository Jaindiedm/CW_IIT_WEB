2. Technical Description

This section outlines the technical implementation details for each page I developed, including HTML structures and CSS styling techniques used.

Feedback Page

For the Feedback Page, I used a combination of HTML, CSS, and JavaScript. The HTML structure includes a simple form with input fields such as text areas for user feedback and a submit button. I used JavaScript for the validation logic, ensuring all required fields are filled before submission. A preview of the user input is displayed once submitted, and a success message is shown once the form is successfully validated.

View the implemented page here: Feedback Page | Validation Results

Key HTML Elements

I implemented the following key HTML elements:

  • <form> for the overall structure of the feedback form.
  • <textarea> for capturing user feedback.
  • <input type="submit"> for submitting the form.

CSS Techniques

For styling, I applied these key techniques:

  • Flexbox for the layout to center the form on the page.
  • Padding and Margin to provide visual separation and structure.
  • Hover effects on buttons for better user interaction.

The purpose behind these choices was to create an engaging, accessible, and responsive feedback form that provides real-time feedback and validation to users.

Team Page

The Team Page introduces each member of the team with details such as their name, role, and responsibilities.

View the implemented page here: Team Page | Validation Results

Key HTML Elements

I implemented the following key HTML elements:

  • <section> to divide the content into team member sections.
  • <h2> and <p> to display the name and role of each team member.

CSS Techniques

For styling, I applied these key techniques:

  • Grid system to ensure each team member's information is evenly spaced.
  • Padding and margin properties to provide visual separation and structure.

The page was designed to be both functional and visually clear, providing users with information about each team member in a structured format.

Content Page

The Content Page was developed to present information about specific topics in a structured format. It includes a mix of text, images, and links to provide detailed content for the user.

View the implemented page here: Content Page | Validation Results

Key HTML Elements

The following key HTML elements were used:

CSS Techniques

I applied the following CSS techniques:

These techniques helped to ensure the page was well-structured and visually appealing to users.

3. Challenges and Lessons Learned

Challenges:

  • Ensuring form validation across multiple pages was consistently working as expected.
  • Making the pages responsive to work seamlessly across various devices.

Lessons:

  • Working with JavaScript to validate user input in real-time was highly rewarding and added dynamic functionality to the pages.
  • Responsive design techniques are crucial for ensuring users on different devices have a good experience on the website.

4. Accessibility

For accessibility, I ensured that:

  • Form elements are clearly labeled for screen readers.
  • All images have proper alt text to make them accessible.
  • Keyboard navigation is supported, particularly on form fields and interactive elements.

5. Compliance

  • Data is handled securely and in compliance with privacy regulations.
  • Pages follow accessibility guidelines for screen readers and keyboard navigation.
  • Responsive design was implemented for mobile and tablet users.
