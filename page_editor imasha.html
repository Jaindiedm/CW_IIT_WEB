Content Readability

Challenge: Ensuring text remains readable against changing backgrounds.

Challenges:

  • Missing closing tags in HTML structure.
  • Proper semantic HTML organization.
  • Accessibility issues such as missing alt text for images.

Solutions

  • i've used the W3C validation tools to identify and fix HTML errors.

  • Implemented proper semantic HTML structure with appropriate section, header, and navigation elements.

CSS Styling Issues

Challenges:

  • Redundant CSS rules causing code bloat
  • Browser compatibility issues with certain CSS properties
  • Maintaining consistent styling across different pages

Solutions

  • Cleaned up redundant CSS rules to improve performance.
  • Implemented a consistent color scheme using variables (var(--neutral-dark))

  • Added hover effects and transitions for better user experience.

4. Accessibility

Ensuring accessibility was a priority throughout my development process. I implemented the following accessibility features for each page:

spalsh screen Accessibility

  • Descriptive Alt Text:Add meaningful alt text to all images to ensure screen reader users understand their purpose.
  • Enhanced Color Contrast: Improve text-to-background contrast ratios to meet WCAG 2.1 AA standards (4.5:1 for normal text, 3:1 for large text).
  • Properly Associated Form Controls: Ensure all form elements have explicitly associated labels using the 'for' attribute.
  • Keyboard Navigation Support: Implement visible focus indicators and logical tab order for all interactive elements.

volunteer page Accessibility

  • Interactive Navigation Elements:It provides visual clickable nodes that link to different pages of the website, making it easier for users to understand the site structure.
  • Focus States: The CSS includes .node:focus styling, which suggests some consideration for keyboard navigation.
  • Simple Color Scheme:This uses a consistent color scheme that maintains visual hierarchy.
  • Visual Relationships:The connecting lines in the SVG clearly show the hierarchical relationship between different pages.
  • Mobile Viewport Setting:This page includes the viewport meta tag for better display on mobile devices.
  • Go-to-Top Feature: There's a "go-to-top" button to help users quickly return to the top of the page.

Content Page Accessibility

  • HTML Language Attribute:The page correctly uses lang="en" in the HTML tag to specify the content language
  • Responsive Design Elements:Media queries to adjust layout for smaller screens.Flexible content container with max-width.
  • Content Organization:Well-structured content with clear headings and sections, lists for presenting information in an organized format and side navigation that outlines the page structure
  • Focus Management:Smooth scrolling behavior for better orientation when navigating
