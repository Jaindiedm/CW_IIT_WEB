2. Technical Description

This section outlines the technical implementation details for each page I developed, including HTML structures and CSS styling techniques used.

User Profile

The User Profile Page was developed using HTML and CSS to create a visually appealing and structured layout. The page includes a profile avatar, user details, skills, goals, and a contact section, ensuring a well-organized presentation of information.

View the implemented page here: User Profile | Validation Page

Key HTML Elements

I implemented the following key HTML elements:

  • <div class="container1"> - Primary container that establishes the page layout grid and sets overall width constraints
  • <img class="profile-picture"> - Displays the user's profile photo with circular styling for visual identity
  • <h1 class="profile-name"> - Presents the user's name in large, prominent text to establish identity
  • <p class="profile-title"> - Shows the user's professional title or role in smaller supporting text
  • <div class="skill-bar"> - Creates the visual container for skill level indicators showing proficiency percentages
  • <textarea> - Creates user input fields for collecting contact information and messages
  • <div class="icon-item"> - Creates containers for activity icons with supporting text descriptions

CSS Techniques

For styling, I applied these key techniques:

  • .container1 - Establishes two-column grid layout with responsive properties for the entire profile.
  • .card - Creates consistent boxed sections with shadows, rounded corners, and hover effects.
  • .profile-section- Centers profile content and applies specific spacing for the identity area.
  • .skill-level - Generates colored bars that visually represent skill proficiency percentages.
  • .icon-list- Creates flexible, wrapping layout for displaying activities and interests with icons.
  • .form-group- Applies consistent spacing and organization to form field groups.
  • .profile-picture- Applies circular cropping, border styling, and sizing to the profile image.
  • .skill-bar- Establishes background track for skill indicators with consistent height and radius.
  • @media (max-width: 768px)- Adjusts layout for mobile devices by converting two columns to one.

The User Profile Page presents user details, skills, goals, and volunteer work with a structured, visually appealing design. Using HTML and CSS, it ensures responsiveness, interactivity, and professionalism through cards, progress bars, and a contact form, enhancing user engagement and accessibility.

Site Map

A sitemap is a structured file or page that outlines the organization and content of a website, serving as a guide for both search engines and users. For users, an HTML sitemap provides a clear overview of the website's structure, making navigation easier, especially on larger websites. Overall, a sitemap improves the discoverability of content and enhances user experience by organizing information in a clear and accessible way.

CSS Techniques

For styling, I applied these key techniques:

  • Global Styling and Reset:
    1. * { margin: 0; padding: 0; box-sizing: border-box; } - Ensures uniform spacing
    2. font-family: Arial, sans-serif; - Sets default typography
  • Layout and Positioning
    1. .video-container { position: fixed; top: 0; left: 0; width: 100%; height: 100%; } - Full-screen video background
    2. .content { position: relative; z-index: 1; text-align: center; } - Ensures readable content above background
  • Typography & Text Styling
    1. h1 { font-family: Cambria, Cochin, Georgia, Times, 'Times New Roman', serif; } - Custom font for headings
    2. text { font-size: auto; font-weight: bold; font-family:Verdana, Geneva, Tahoma, sans-serif; } - SVG text styling
  • Colors & Backgrounds
    1. body { background: #f4f4f4; } - Light gray background for contrast
    2. h1 { background-color: #85a869; color: #132404; border-radius: 12px; } - Stylish heading with rounded edges
  • Flexibility & Responsiveness
    1. svg { width: 90%; max-width: 800px; } - Adjusts size for different screen sizes
  • Hover & Interactive Effects
    1. .node:hover, .node:focus { fill: #85a869; } - Changes node color on hover
  • Transitions & Animations
    1. .node { transition: fill 0.3s ease; } - Smooth transition on hover effects
  • Z-Index for Layering
    1. .video-container { z-index: -1; } - Keeps video in the background
    2. .content { z-index: 1; }Ensures content remains on top
  • Border & Stroke Styling
    1. h1 { border-radius: 12px; } - Rounded corners for headings
    2. .node { stroke: #cbe5bf; stroke-width: 4; } - SVG node border styling

3. Challenges and Lessons Learned

Responsive Design Challenge

Challenge:

Making the profile page work on different devices and screen sizes.

Solution

  • I implemented a media query for screens under 768px: @media (max-width: 768px) that changes the layout to a single column.
  • i used percentages for width
  • Include proper viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">

Visual Hierarchy

Challenge:

Creating an appealing and intuitive user interface that highlights important information.

Solution

  • Use a two-column grid layout with a clear content organization strategy.
  • Rounded corners with border-radius: var(--border-radius) for a modern look

Navigation and Site Integration

Challenge:

Integrating the profile page with the rest of the site and providing easy navigation.

Solution

  • Comprehensive navigation bar with links to other site sections.
  • Consistent footer with navigation links repeated.
  • "Go to Top" button <a href="#" id="go-to-top"▲"/a> for easy page navigation

Content Accessibility and Navigation

Challenge:

Users might find it difficult to navigate through extensive forestry information.

Solution

I Implemented a fixed side navigation bar that stays visible while scrolling, with direct links to specific sections like "Sustainable Forestry Practices" and "Reforestation Benefits."

Information Organization

Challenge:

Presenting complex forestry concepts in a digestible format.

Solution

Used a clear hierarchical structure with well-defined headings, descriptive paragraphs, and bulleted lists to break down complex topics like sustainable forestry practices and reforestation benefits.

Content Readability

Challenge:

Ensuring text remains readable against changing backgrounds.

Solution

Applied appropriate color contrasts, especially in the transparent boxes where you've used light text on dark backgrounds and dark text on light backgrounds for headings.

Visual Integration with Site Theme

Challenge:

Making the sitemap consistent with the overall site design.

Solution

i've maintained the "Life on Land" (SDG 15) theme with green color palette (#374e32, #85a869, #cbe5bf) and use a environmental video background.

Mobile Responsiveness

Challenge:

Ensuring the sitemap displays well on different screen sizes.

Solution

i've used setting the SVG to display at 90% width with a max-width of 800px and implementing flexible sizing with percentage-based widths.

HTML Structure and Validation Errors

Challenges:

  • Missing closing tags in HTML structure.
  • Proper semantic HTML organization.
  • Accessibility issues such as missing alt text for images.

Solutions

  • i've used the W3C validation tools to identify and fix HTML errors.
  • Added descriptive alt attributes to images to improve accessibility.
  • Implemented proper semantic HTML structure with appropriate section, header, and navigation elements.

CSS Styling Issues

Challenges:

  • Redundant CSS rules causing code bloat
  • Browser compatibility issues with certain CSS properties
  • Maintaining consistent styling across different pages

Solutions

  • Cleaned up redundant CSS rules to improve performance.
  • Implemented a consistent color scheme using variables (var(--neutral-dark))
  • Created responsive layouts using flexbox (display: flex, flex-direction: row-reverse)
  • Added hover effects and transitions for better user experience.

5. Compliance

All pages I developed comply with Janet regulations for web page publication through the following measures:

  • Security Standards: Implementation of secure form handling with proper validation to prevent injection attacks
  • Privacy Protection: Clear privacy notices on forms collecting personal information, with explicit consent mechanisms
  • Intellectual Property: Proper attribution for all third-party content and resources used on the pages
  • Regulatory Requirements: Adherence to UK data protection requirements for form submissions
  • Accessibility Standards: Compliance with WCAG 2.1 AA standards as required by Janet regulations
  • Responsive Design: Implementation of responsive design principles to ensure content is accessible across devices

Specific compliance measures for the volunteer form include:

  • Clear statements about how user data will be processed and stored
  • Secure form submission with appropriate data validation
  • Option to opt out of marketing communications
  • Accessible form design that meets Janet standards for online forms
