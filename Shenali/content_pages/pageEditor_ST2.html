2. Technical Description

This section outlines the technical implementation details for each page I developed, including HTML structures and CSS styling techniques used.

Home Page

The Home Page serves as the introduction to the website, featuring a hero section with a full-width background image, a navigation bar, and an overview of key initiatives. It uses Flexbox for layout, ensuring responsiveness and a visually engaging design.

View the implemented page here: Home Page | Validation Results

Key HTML Elements

I implemented the following key HTML elements:

  • header
    The header element is used to define the header section of the webpage, which typically contains navigation links and branding. In the home page, it includes the navigation bar with links to different pages, as well as the logo and title for the site.
  • section
    The section element is used to group content related to a specific topic or theme. On the home page, it is used to structure the content into sections like "Why Life on Land Matters?" and "Sustain Life on Land," making the page more organized and accessible to users and search engines.
  • Anchor Link
    The a tag is used for creating hyperlinks, allowing users to navigate between pages. In the home page, it is used in the navigation menu and in the cards/tiles section to provide links to other pages such as "Volunteer," "Data," and "Feedback." These anchor links help create a seamless browsing experience across the site.

CSS Techniques

For styling, I applied these key techniques:

  • #panda-image(Main Image Styling)
    The main image is styled to cover the entire width of the page and be responsive. The object-fit: cover; ensures the image maintains its aspect ratio and fills the space without distortion.
  • .image-container::after (Blurred Bottom Effect)
    This style applies a linear gradient effect at the bottom of the image with a slight blur. This effect creates a fading transition that enhances the visual appeal and ensures that the text overlay is more readable.
  • .mission-section(Mission Section Flexbox Layout)
    The mission section uses flexbox to create a responsive layout where content like the tagline and description are displayed side by side. This section has a background gradient, padding, and subtle box shadows for a clean and modern look.
  • .card(Tile/Card Styling)
    Each card or tile is styled with a background image, rounded corners, and a hover effect. The :hover pseudo-class is used to darken the background and reveal text when the user hovers over each card, improving interactivity.

The purpose behind these choices was to create an engaging, accessible, and responsive entry point to our website that clearly communicates the site's purpose while providing easy navigation options.

3. Challenges and Lessons Learned

Challenge: Responsive Design Adjustments

Ensuring all elements, including images and text, scale properly across different screen sizes was difficult.

Solution:

Used flexbox and grid layouts for flexible positioning and applied clamp() for font sizes to adjust text dynamically.

Challenge: Hover Effects on Content Tiles

Initially, the hover effects on the cards were not working smoothly, and the text visibility was inconsistent.

Solution:

Implemented CSS transitions for smooth effects and used opacity changes to reveal text only on hover.

4. Accessibility

Ensuring accessibility was a priority throughout my development process. I implemented the following accessibility features for each page:

Home Page Accessibility

  • Semantic HTML – Used "header, nav, main, and footer" for better screen reader navigation.
  • High Contrast Text – Ensured text readability against background images using proper color contrast.
  • Keyboard Navigation – Made navigation links and interactive elements accessible via the keyboard.

Table Page Accessibility

  • ARIA Labels – Used aria-label and aria-describedby for better screen reader interpretation of tables.
  • Keyboard-Friendly Table Navigation – Ensured users can navigate through table rows and columns using the keyboard.
  • Alternative Text for Images – Provided descriptive alt attributes for all images related to tabular data.

Content Page Accessibility

  • Proper Heading Structure – Used hierarchical headings (h1, h2, etc.) to organize content for better screen reader interpretation.
  • Sufficient Color Contrast – Ensured contrast between text and background meets WCAG standards.
  • Accessible Links & Buttons – Styled links with clear hover states and added aria-labels where necessary.

5. Compliance

All pages I developed comply with Janet regulations for web page publication through the following measures:

  • Security Standards: Implementation of secure form handling with proper validation to prevent injection attacks
  • Privacy Protection: Clear privacy notices on forms collecting personal information, with explicit consent mechanisms
  • Intellectual Property: Proper attribution for all third-party content and resources used on the pages
  • Regulatory Requirements: Adherence to UK data protection requirements for form submissions
  • Accessibility Standards: Compliance with WCAG 2.1 AA standards as required by Janet regulations
  • Responsive Design: Implementation of responsive design principles to ensure content is accessible across devices

Specific compliance measures for the volunteer form include:

  • Clear statements about how user data will be processed and stored
  • Secure form submission with appropriate data validation
  • Option to opt out of marketing communications
  • Accessible form design that meets Janet standards for online forms
